# CLAUDE.md

## Project Overview

디지털 문헌학 연구: 이노우에 데츠지로 『철학과 종교』(1915) × 이돈화 『인내천요의』(1924) 텍스트 관계 분석. 서울대 국사학과 허수 교수 연구 프로젝트.

## Commands

```bash
pip install -r app/requirements.txt   # 의존성 설치
streamlit run app/app.py              # 발표용 웹앱 실행
```

### Data Structure

- 1915: `C##-P##-S##` (장-문단-문장), C00~C28 (29개 장: 서장 1 + 본문 26 + 부록 2)
- 1924: `C##-S##-I##-P##-S##` (장-절-항-문단-문장), C01~C06 (6개 장)

## Coding Guidelines

1. **역할**: 역사학적 맥락을 이해하는 연구 조교
2. **언어**: 한국어, 정중하고 학술적인 어조
3. **기술**: Python 3.11, Streamlit, Pandas, Plotly
4. **원칙**: 사료 원문 변형 최소화

## Research Context

**분석의 3대 축**:
1. 지식의 차용: 과학적 권위의 투명한 수용
2. 논리의 내재화: '철학' 개념의 소거와 주체 교체
3. 가치의 전복: 제국주의 논리의 해체와 재구성

**분석 방법**: 한자어 토큰화 (`r'[一-龥]{2,}'`), Jaccard 유사도, 문단 단위 비교

### 빈도 카운트 기준

- **포함**: `line_class`가 `TEXT`, `STRUCT`인 행
- **제외**: `RTC_TEXT`, `ANNOTATION`, `structure_id`가 `TOC`/`ROOT`

**완료된 작업**:
1. 현재 웹앱 스토리라인 점검 완료
2. 장절별 논지 중심 스토리라인 정리 → `docs/presentation-storyline.md` 저장
3. 폴더 구조 단순화 완료 (DB 통합, 빈 디렉토리 정리, 구버전 삭제)
4. 발표문 텍스트를 `app/sections/` 마크다운 파일로 분리 완료 (10개 .md 파일)
5. app.py를 `render_section()` + `<!-- component: name -->` 마커 기반 구조로 리팩터링 완료
6. 테이블 데이터를 `app/tables.json`으로 분리 완료 (20개 테이블)
7. 미사용 파일 `삭제검토대상/`으로 이동 완료
8. 웹앱 레이아웃 변경 완료: 사이드바 라디오 버튼 → HTML 앵커 목차, 본문 한 페이지 스크롤 방식으로 전환, 사이드바 너비 240px 축소
9. 스토리라인 논리 검토 및 수정 완료:
   - 팩트 오류 수정: 1924 문단 수 366→365 (전체 문서 8곳)
   - C03-S04 참조쌍 수 재검증 완료 (발표문 수치 정확 확인)
   - Ⅲ→Ⅳ 전환 동기 부여 문장 추가 ('모듈' → '기표 소거' 연결)
   - Ⅳ-1 역상관 주장 보강: 반론 선제 제시 + 대조군(내용어 vs 범주어) 추가
   - Ⅳ-2 대체 패턴 사례 수(4건) 한계 명시, 인과적 어조 완화
10. 스토리라인 + 발표문 가독성 개선 완료 (대학 3학년 문과생 수준 목표):
    - 전문 용어 풀이: 기표→단어, 전유→받아들이다, 이데올로기적 모듈→이론적 도구, 탈각→걷어냄, 변증법→풀어쓰기
    - 절 간 연결고리 5곳 추가 (각 절 끝에 다음 절로 이어지는 질문/문장)
    - 절 제목 변경: Ⅲ-1/Ⅲ-2/Ⅳ-1/Ⅳ-2/Ⅳ-3 (내용을 직관적으로 드러내는 제목으로)
    - 설명·비유 추가: 현상즉실재론 한 줄 정의, 국체론적 철학 설명, '범용 부품' 비유, 자카드 유사도 직관적 설명
11. 머리말 내용 보강 및 각주 추가:
    - 이노우에 서술 정교화: "정립하고자"→"활용하고자" (현상즉실재론은 독자적 철학 이론, 국체론에 동원된 것)
    - 디지털 문헌학 개념 설명 추가: 전통적 문헌학 + 컴퓨터 대규모 분석, 동아시아 학계 동향 언급
    - HTML 각주 4개 신설 (머리말 하단, `<sup>` + `<div>` 방식)
12. data 폴더 구조 재편 및 코퍼스 구축 과정 기술
13. 표·그림 넘버링 및 본문 참조 구체화 (표 1~18, 그림 1~4)
14. Ⅱ-1, Ⅱ-2 본문 수정 (가독성, types/tokens 설명 등)
15. Ⅳ-1 수정: freq_compare_table 일본어 기능어 제외, 해석적 주장→사실 기술로 완화
16. Ⅳ-2 전면 재구성 → Ⅳ-2 + Ⅳ-3 분리:
    - **Ⅳ-2 (원문에서 읽는 대체 패턴)**: 소거 사례 ①② + 대조군 + 사례 ③(위양성, 확인 편향 경고)
    - **Ⅳ-3 (패턴의 검증: 111쌍에서 전체 코퍼스로)**: 차등 어휘 분석 + 의미장 분석 + 종합
    - 사례 ①: '哲學的價値/哲學系統' → '哲人+眞理' (패턴 A)
    - 사례 ②: '哲學宗敎의 特色' → '理想의 境涯' (패턴 B)
    - 사례 ③: C14-S03-P04(Rank 10, 공통 9종)이 실제 원천 → 확인 편향 경고
    - 대조군: 불교 교리의 철학적 성격 맥락에서는 哲學 유지
    - 차등 어휘: A-B 430종(哲學 복합어 8종), B-A 326종(哲人·眞理 부상)
    - 의미장: 哲學 공기어 19종(배율≥1.5x), 1924 매칭 26문단 중 92.3%에서 哲學 부재
    - 새 테이블: diff_vocab_table(표 17), phil_compound_table(표 18)
17. Ⅲ-1에 참조쌍 네트워크 그래프(그림 2) 추가
18. Ⅴ 맺음말 업데이트: 세 층위의 정량적 데이터(원문 대조 + 차등 어휘 + 의미장) 반영
